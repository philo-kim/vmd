# Code Of Conduct

VMD should be a useful and respectful open project.

Participants are expected to:

- be direct and constructive
- focus criticism on ideas and implementation
- give enough context for technical claims
- avoid harassment, insults, and personal attacks
- respect that this is an early experimental format

Maintainers may remove issues, comments, or pull requests that are abusive,
spammy, or unrelated to the project.

The goal is a practical environment for designing a shared visual document
format.
